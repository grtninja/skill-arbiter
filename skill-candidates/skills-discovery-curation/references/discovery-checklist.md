# Skill Discovery Checklist

1. List local installed skills and any blacklisted entries.
2. Route the work through `skill-hub` and capture the recommended chain.
3. For multi-repo operators, run `skills-cross-repo-radar` and capture the latest JSON report.
4. If the request is about recent Codex work, inspect recent `.codex/workstreams` and continuity artifacts in addition to repo history.
5. Include `skill-common-sense-engineering` when avoidable mistakes or hygiene regressions recur.
6. Include `skill-auditor` when new/updated skills are in scope.
7. Include `skill-enforcer` when policy alignment is required across repos.
8. Identify repeated tasks in target repo or workspace (release, CI triage, policy checks, packaging, contract alignment, startup acceptance, etc.).
9. Match tasks to existing skills before proposing new ones.
10. For each candidate, record `skill-auditor` classification (`unique` vs `upgrade`) and nearest peer.
11. Admit candidates one-by-one with `skill-arbiter` personal-lockdown mode.
12. Require arbiter status `pass` before marking candidates admitted.
13. Keep only proven skills pinned immutable.
