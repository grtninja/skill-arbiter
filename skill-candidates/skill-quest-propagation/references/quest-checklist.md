# Quest Checklist

## Before Execution

- prior quest registry rebuilt
- objective named in one sentence
- ordered lanes declared
- state file path chosen
- stop conditions written down

## During Execution

- only one lane marked `in_progress`
- each lane transition recorded
- artifact paths captured as they are created
- next step updated after every interruption or redirect
- quest registry rebuilt after meaningful state changes

## After Routing Changes

- branch alias sidecar regenerated
- skill alias sidecar regenerated
- new lane is visible where the operator expects it

## Handoff Minimum

- quest title
- current lane
- next concrete action
- main evidence paths
- registry paths for the historical quest inventory
